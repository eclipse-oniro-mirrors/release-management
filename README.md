SIG Release 的主要工作职责是在社区协调各SIG的Leader、QA等各个团队，完成OpenHarmony社区版本的发布工作。主要职责如下：

- 规划和计划OpenHarmony版本的发行时间表
- 在开发/测试周期中跟踪（更新updates或功能feature）的开发状态
- 发布协调（在QA，发行经理，SIG等范围内），参与相关SIG组和发布相关等会议
- 负责OpenHarmony项目的交付过程协调
